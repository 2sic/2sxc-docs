    Debugging 2sxc using Insights

    2sxc has powerfull debugging built in. We call it 2sxc Insights and it's available for both the Server and the JavaScript-UI and API.

    Since we're always a bit short on documentation resources, here's just the short version:

    Debugging Server Side Stuff using Insights

    Every module that is rendered and every api-call that is made is logged extensively. This is a life-saver both for finding issues in 2sxc, but also for find out where your settings went wrong.

    Since keeping these logs would clutter up memory, only 500 requests are stored, then logging is paused. As a super-user, you can always re-activate the recording for another 500 requests.

    Debugging Client-Side JavaScripts, APIs or Custom Toolbars

    This was introduced in 2sxc 10.27. There are two things you need to know:

    1. Everything that 2sxc does internally is automatically logged.
    2. If you add ?debug=true to the url, the logging is intensified.
