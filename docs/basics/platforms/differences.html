    Razor API

    Collection Feature Dnn Oqtane Compatibility Notes / Alternatives
    Razor APIs
    Razor @helper ✅ ⛔ not in .net core 5 create a separate file for each helper and use RenderPage
    Razor Dnn object ✅ ⛔ DNN only Use CmsContext, a bit more limited. For Oqtane features use DependencyInjection.
    Razor Koi static object ✅ ⛔ Not supported Use Dependency Injection version of Koi 2
    Razor Koi Service ✅ ✅ core functionality Unclear if the full functionality will be re-implemented, as it had some design flaws

    Features missing in Oqtane

    1. @helper Razor Syntax is missing in Oqtane because of .net Core 5 which doesn't support it.
      Recommendation:
    2. The object Dnn is missing for obvious reasons. All important properties will be on CmsContext which will work across platforms
    3. Koi is being implemented to use Dependency-Injection, the old global static variable approach will become obsolete and not work on Oqtane

    Undergoing Research

    1. Unclear if RenderPage works or should be re-implemented?
    2. RazorBlade should work but isn't tested yet

    WIP

    1. Link.To(...) isn't implemented yet, and it could have some issues because Oqtane always has a base-ref html-header tag

    Features implemented / done

    1. Razor Base class - you should use ToSic.Custom.Razor12 as a base class
    2. Razor Code-Behind - probably done, should verify

    Partially Implemented features

    1. Koi supports the CSS Information API, but not yet the class-generating API. We're not sure if we will implement this.

    Not yet Implemented features

    1. CustomizeData / CustomizeSearch are not implemented yet, we're not yet sure how we want to implement this
    2. RazorBlade feature to change page title or set headers etc. cannot work yet

    WebAPIs

    1. A new base class has been defined but not fully tested
    2. Decorator Attributes must still be defined as they should work across platforms despite differences
